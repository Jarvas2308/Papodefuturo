# AGENTS

1. Ler `README.md` e `AGENTS.md` antes de alterar o código.
2. Implementar apenas o escopo solicitado.
3. Não adicionar banco, Supabase ou APIs sem solicitação explícita.
4. Não implementar cálculos financeiros dentro de componentes visuais.
5. Manter regras de negócio separadas da interface.
6. Evitar arquivos grandes e responsabilidades misturadas.
7. Manter toda a interface em português do Brasil.
8. Executar lint e build antes de concluir.
9. Não avançar espontaneamente para outras etapas.
10. Não alterar decisões de produto silenciosamente.
11. Usar npm e manter o `package-lock.json`.
12. Não inserir segredos, chaves ou credenciais no repositório.
